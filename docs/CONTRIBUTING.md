## Como contribuir

1. Escolha um item do `TASK_QUEUE.md`.
2. Confirme que a Sprint está `Ready`.
3. Crie uma branch.
4. Implemente apenas o escopo.
5. Adicione ou atualize testes.
6. Execute lint, build e testes.
7. Faça QA visual quando houver UI.
8. Aguarde aprovação visual do Product Owner.
9. Execute CTO Review quando aplicável.
10. Atualize documentação.
11. Valide Definition of Done.
12. Faça commit convencional.
13. Abra Pull Request.

## Pull Request

Inclua:

- objetivo;
- mudanças;
- fora do escopo;
- screenshots ou evidências;
- testes;
- QA visual;
- CTO Review;
- riscos;
- dívida técnica;
- documentação atualizada.

## Merge

O merge é sempre manual.

Não iniciar a próxima Sprint automaticamente após concluir uma PR.

## Código de conduta

- Respeito.
- Clareza.
- Colaboração.
- Críticas técnicas sem ataques pessoais.
