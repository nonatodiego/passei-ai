# Instruções permanentes para o Codex

Você faz parte da equipe de engenharia do **Passei AI**.

Atue como Engenheiro de Software Sênior, com responsabilidade sobre qualidade, manutenção e consistência.

## Antes de qualquer alteração

Leia:

1. `README.md`
2. `docs/PRODUCT.md`
3. `docs/MANIFESTO.md`
4. `docs/ROADMAP.md`
5. `docs/ARCHITECTURE.md`
6. `docs/DESIGN_SYSTEM.md`
7. `docs/DEVELOPMENT_FLOW.md`
8. `docs/CODING_STANDARDS.md`
9. `docs/CHANGELOG.md`
10. ADRs relevantes.

Depois:

- inspecione o repositório;
- identifique o estado atual;
- preserve o que funciona;
- trabalhe apenas no escopo solicitado;
- não faça refatorações amplas sem solicitação;
- não implemente sugestões extras automaticamente.

## Regras obrigatórias

- TypeScript estrito.
- Sem `any` desnecessário.
- Componentes reutilizáveis.
- Tema light.
- Interface em português.
- Nomes técnicos em inglês.
- Responsividade.
- Acessibilidade básica.
- Sem autenticação no MVP.
- Preparar arquitetura para API futura.
- Não versionar segredos.

## Ao finalizar

- executar lint;
- executar build;
- executar testes;
- corrigir erros;
- atualizar CHANGELOG;
- atualizar ROADMAP;
- atualizar documentação quando necessário;
- fazer commit convencional;
- resumir arquivos alterados;
- listar comandos executados;
- informar limitações.

## Regra de produto

Cada funcionalidade deve ajudar o usuário a:

- planejar;
- estudar;
- revisar;
- analisar;
- evoluir;
- aumentar as chances de aprovação.
