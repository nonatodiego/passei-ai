# ADR-005 — Autenticação no MVP

## Status
Aceito.

## Decisão
Não implementar autenticação no MVP.

## Motivo
- Uso pessoal inicial.
- Reduzir complexidade.
- Validar produto antes de multiusuário.

## Consequências
- Usar perfil local padrão.
- Arquitetura deve permitir autenticação futura.
- Não assumir usuário único nas regras de domínio definitivas.
