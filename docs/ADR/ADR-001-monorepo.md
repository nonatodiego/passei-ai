# ADR-001 — Monorepo

## Status
Aceito.

## Decisão
Usar monorepo com frontend, backend, docs e scripts no mesmo repositório.

## Motivo
- MVP.
- Projeto pequeno.
- Documentação centralizada.
- Integração simples.
- Melhor experiência para contribuição.

## Consequências
- CI deve identificar áreas alteradas.
- Convenções devem ser claras.
