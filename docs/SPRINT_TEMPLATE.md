## Definition of Ready

- [ ] Objetivo claro.
- [ ] Problema de negócio descrito.
- [ ] Escopo definido.
- [ ] Fora do escopo registrado.
- [ ] Critérios de aceite definidos.
- [ ] Riscos identificados.
- [ ] Dependências disponíveis.
- [ ] Sem bloqueios críticos.
- [ ] Branch definida.
- [ ] Documentação obrigatória lida.
- [ ] Sprint marcada como `Ready` no `TASK_QUEUE.md`.

## Definition of Done

- [ ] Escopo implementado.
- [ ] Critérios de aceite atendidos.
- [ ] Lint aprovado.
- [ ] Build aprovado.
- [ ] Testes aprovados.
- [ ] QA visual aprovado pelo Product Owner.
- [ ] Acessibilidade básica verificada.
- [ ] Responsividade validada.
- [ ] CTO Review realizada.
- [ ] Dívida técnica registrada.
- [ ] CHANGELOG atualizado.
- [ ] ROADMAP atualizado.
- [ ] PROJECT_METRICS atualizado quando necessário.
- [ ] QUALITY_SCORE atualizado.
- [ ] TASK_QUEUE atualizada.
- [ ] Commit convencional criado.
- [ ] Sem arquivos temporários ou segredos.
- [ ] Product Owner aprovou a entrega.
